Phase 1: Data Preparation and Feature Engineering
1.1 Data Acquisition

Daily dengue patient counts were collected from DGHS for all eight administrative divisions of Bangladesh (2022‚Äď2025). Corresponding meteorological variables (temperature, rainfall, humidity, wind speed, dew point, etc.) were acquired from public APIs and ERA5 reanalysis datasets.

1.2 Advanced Feature Engineering

A total of ~47 temporal and climatic predictor variables were engineered, including:

Autoregressive lags using shift(lag)

Rolling window features (mean, std, min, max) using shift(1) to avoid leakage

Fourier seasonal components for periodic patterns

Derived climate indices (heat index, humidity index, rainfall anomalies)

Spatial and seasonal encodings (division ID, monsoon label)

1.3 Leakage Prevention and Data Integrity Measures

To ensure strict time-series validity:

All temporal features used shift-based operations (lagged only from past).

Log-transform of the target and feature scaling were applied only after splitting the dataset.

No information from the Test period (2025) leaked into the Training period (2022‚Äď2024).

These steps ensure unbiased and reliable forecasting performance.
